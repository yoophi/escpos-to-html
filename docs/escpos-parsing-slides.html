Current implementation

ESC/POS 파싱 방식

현재 파서는 ESC/POS 입력 문자열을 바이트 배열로 정규화한 뒤, 한 번의 상태 머신 순회로 영수증 라인, 텍스트 스타일, 프린터 제어 이벤트, 경고를 생성합니다. HTML 렌더링은 이 중간 데이터 모델만 소비합니다.

대상: packages/escpos 입력: escaped 또는 hex 출력: ParseResult, HTML 문서 위치: docs/*.html

Pipeline

입력에서 HTML까지의 처리 흐름

1 Input

사용자가 escaped 텍스트 또는 hex 바이트 문자열을 입력합니다.

2 Bytes

parseInput이 입력을 숫자 바이트 배열로 변환합니다.

3 State machine

parseEscpos가 명령 바이트와 일반 텍스트를 순차 해석합니다.

4 ParseResult

라인, span, 이벤트, 경고, 원본 바이트를 중간 데이터로 보관합니다.

5 HTML

renderHtml이 데이터 모델을 영수증 마크업으로 변환합니다.

Input decoding

입력 문자열은 먼저 바이트로 정규화합니다

escaped 모드

일반 문자는 UTF-8로 인코딩하고, \xNN, \uNNNN, \n, \r, \e, \g 같은 이스케이프를 제어 바이트로 변환합니다.

명령 구분 공백

\e a\x01처럼 ESC 또는 GS 뒤에 사람이 읽기 좋게 넣은 공백과 탭은 건너뜁니다. 결과 바이트는 1B 61 01입니다.

hex 모드

입력 전체에서 2자리 hex 쌍만 추출합니다. 공백, 콜론, 하이픈 같은 구분자는 무시되고 홀수 nibble은 버려집니다. 

parseInput("\\e a\\x01동네분식\\n", "escaped")

=> [
  0x1B, 0x61, 0x01,
  0xEB, 0x8F, 0x99, ...
  0x0A
]
중요한 전제

파서는 프린터와 동일하게 바이트 스트림을 기준으로 동작합니다. 사용자가 작성한 \e a의 a는 텍스트가 아니라 ESC/POS 정렬 명령 바이트입니다.

Intermediate model

HTML 전에 생성하는 중간 데이터

type ParseResult = {
  lines: ReceiptLine[]
  events: ControlEvent[]
  warnings: string[]
  bytes: number[]
}

type ReceiptLine = {
  align: "left" | "center" | "right"
  spans: ReceiptSpan[]
}

type ReceiptSpan = {
  text: string
  style: TextStyle
}
lines

프린터가 출력할 텍스트 라인입니다. 각 라인은 정렬값과 여러 텍스트 span으로 구성됩니다.

events

컷, 드로어 펄스, 비프, 피드처럼 화면 텍스트가 아닌 프린터 제어 명령입니다.

warnings

지원하지 않는 명령, 부족한 인자, 무시한 제어 문자를 offset과 함께 기록합니다.

bytes

입력에서 디코딩된 원본 바이트입니다. 디버깅용 hex 표시의 기준 데이터입니다.

Parser state

한 번의 순회로 상태를 갱신합니다

유지하는 상태
  • style: 굵게, 밑줄, 반전, 확대, 폰트 A/B/C
  • align: left, center, right
  • textBuffer: 아직 span으로 확정하지 않은 텍스트 바이트
  • lines, events, warnings
flush 기준

ESC 또는 GS 명령을 만나면 먼저 텍스트 버퍼를 span으로 확정합니다. 같은 스타일의 인접 span은 합쳐서 불필요한 노드를 줄입니다. LF는 현재 라인을 마감하고 새 라인을 만듭니다. 

for each byte:
  LF        -> newLine()
  CR        -> ignore
  BEL       -> flushText(), event(beep)
  ESC / GS  -> flushText(), parse command
  control   -> warning
  otherwise -> textBuffer.push(byte)
라인 정리

순회가 끝나면 마지막 텍스트를 flush하고, 끝에 남은 빈 라인은 제거합니다. 따라서 컷 명령 뒤의 불필요한 빈 줄이 HTML에 남지 않습니다.

Command coverage

현재 지원하는 ESC/POS 명령

범주 명령 처리 결과
Text LF, CR, BEL, 일반 바이트 LF는 새 라인, CR은 무시, BEL은 비프 이벤트, 일반 바이트는 텍스트 버퍼에 누적합니다.
ESC @, E, -, a, M 초기화, 굵게, 밑줄, 정렬, 폰트 선택을 현재 style 또는 align에 반영합니다.
Event ESC d, ESC p 피드 라인을 추가하고 feed 이벤트를 기록하거나, 금전함 펄스를 drawer 이벤트로 기록합니다.
GS !, B, V 텍스트 확대, 반전, 컷 이벤트를 처리합니다. GS V 41 n은 feed 포함 컷으로 기록합니다.
Warning 미지원 명령, 부족한 인자, 기타 제어 문자 파싱을 가능한 만큼 계속하면서 offset이 포함된 한국어 경고 메시지를 남깁니다.

HTML rendering

HTML 렌더러는 ParseResult만 사용합니다

기본 구조

루트는 .escpos-receipt이고, 각 라인은 .receipt-line과 .align-left, .align-center, .align-right 클래스를 갖습니다.

span 출력 옵션

wrapPlainTextSpans가 true이면 일반 텍스트도 span으로 감쌉니다. false이면 기본 스타일 텍스트는 span 없이 출력하고, 스타일이 있는 텍스트만 span으로 유지합니다. 

<div class="escpos-receipt">
  <div class="receipt-line align-center">
    <span style="font-weight: 800">동네분식</span>
  </div>
  <div class="receipt-line align-right">
    합계 19,500
  </div>
</div>

Korean width

한글 고정폭 처리는 프리뷰 계층에서 보정합니다

데이터 모델은 원문 유지

ReceiptSpan.text는 디코딩된 문자열을 그대로 보관합니다. 파서 단계에서 한글을 공백으로 확장하지 않기 때문에 검색, 복사, HTML escape가 단순하게 유지됩니다.

표시 폭 계산

isWidePrintChar는 Hangul, CJK, fullwidth 범위를 2칸 문자로 판단합니다. 앱의 편집기와 프리뷰는 이 정보를 사용해 영수증 프린터의 고정폭 레이아웃에 가깝게 표시합니다. 

isWidePrintChar("A")  // false, 1 column
isWidePrintChar("가") // true, 2 columns
isWidePrintChar("漢") // true, 2 columns
isWidePrintChar("！") // true, 2 columns
역할 분리

파서 패키지는 명령 해석과 데이터 생성을 담당합니다. 실제 화면의 column 정렬, 텍스트 측정, 영수증 폭 보정은 웹 앱의 렌더링 계층에서 처리하는 구조입니다.

Quality gates

현재 테스트가 검증하는 범위

입력 디코딩

escaped, unicode, 제어 이스케이프, invalid escape, hex pair 추출을 검증합니다.

파서 동작

정렬, 굵게, 밑줄, 폰트, 확대, 반전, 피드, 컷, 드로어, 비프, 경고, 라인 trim을 검증합니다.

렌더링

HTML escape, alignment class, inline style, 기본 텍스트 span 표시 옵션을 검증합니다.
